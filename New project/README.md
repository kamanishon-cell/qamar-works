# UniTask

UniTask - учебный MVP менеджера задач для студентов. Проект написан на Python и запускает веб-интерфейс с календарем, списком задач, важностью, повторами, аккаунтами пользователей и напоминаниями.

## Возможности

- регистрация и вход в личный аккаунт;
- отдельные задачи, календарь и уведомления для каждого пользователя;
- добавление задач, экзаменов и проектных работ;
- отметка источника: вручную или университет;
- важность задачи: высокая, обычная, низкая;
- повтор: единожды, каждый день, будние дни, выбранные дни;
- сортировка списка: по времени, по важности, недавно добавленные;
- календарный вид и обычный список;
- напоминание до дедлайна или в момент дедлайна;
- повторное напоминание каждые 30 минут, если студент не ответил.

## Публичный сайт

Чтобы сайт был доступен всем из интернета, его нужно разместить на хостинге. Просто запустить сервер на ноутбуке недостаточно: адрес `127.0.0.1` работает только у тебя, а адрес `192.168.x.x` работает только внутри одной Wi-Fi сети.

Для публикации проект уже подготовлен:

- `Procfile` - команда запуска для хостинга;
- `requirements.txt` - файл зависимостей;
- `render.yaml` - конфигурация для Render;
- `app.py` читает порт из переменной `PORT`, как обычно требуют хостинги.

## Вариант 1: Render

Render подходит для простого публичного демо. По официальной документации Render веб-сервис должен слушать `0.0.0.0`, а порт можно задавать через переменную `PORT`; по умолчанию Render ожидает порт `10000`.

Шаги:

1. Создай аккаунт на Render.
2. Загрузи проект на GitHub.
3. В Render нажми `New` -> `Web Service`.
4. Подключи GitHub-репозиторий с проектом.
5. Укажи:

```text
Build Command: pip install -r requirements.txt
Start Command: python app.py
```

6. Нажми `Deploy`.
7. После деплоя Render даст публичную ссылку вида:

```text
https://unitask.onrender.com
```

Эту ссылку можно отправлять любым пользователям.

## Вариант 2: Railway

Railway тоже подходит для публикации Python-приложения. По официальной документации приложение должно слушать `0.0.0.0:$PORT`; Railway сам выдаёт переменную `PORT`.

Шаги:

1. Создай аккаунт на Railway.
2. Загрузи проект на GitHub.
3. В Railway создай новый проект из GitHub-репозитория.
4. Укажи start command:

```text
python app.py
```

5. В настройках Networking нажми `Generate Domain`.
6. Railway выдаст публичный адрес сайта.

## Вариант 3: Быстрый временный доступ

Для временной демонстрации можно использовать HTTPS-туннель, например Cloudflare Tunnel или ngrok. Тогда сайт запускается на твоем компьютере, а сервис выдает публичную ссылку. Минус: если выключить компьютер или закрыть туннель, сайт перестанет работать.

## Важное ограничение SQLite

Сейчас данные хранятся в:

```text
data/unitask.db
```

Для учебного MVP этого достаточно. Но на некоторых бесплатных хостингах файловая система может очищаться при перезапуске или деплое. Для настоящего продукта лучше заменить SQLite на PostgreSQL. Логика аккаунтов уже сделана так, что позже базу можно перенести.

## Push-уведомления

Внутренний блок "Уведомления" работает на любом хостинге. Системные браузерные push-уведомления лучше работают на публичном HTTPS-адресе, поэтому хостинг с `https://...` подходит лучше, чем запуск по локальному IP.
